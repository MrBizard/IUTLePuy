Membre _tWinMain (HINSTANCE hInstance, HINSTANCE hPrevInstance, LPTSTR lpCmdLine, int nCmdShow)

1) Créer 4 objets de types CCase A, B, C et D disposées en carré au commentaire //A faire 1) dans la fonction _tWinMain()
La première ligne est A - B, la seconde ligne est C - D.
Les connexions entre les cases doivent être : B à droite de A, D en dessous de B, C à gauche de D.

2) Afficher les quatres cases A, B, C, D au commentaire //A faire 2) dans la fonction _tWinMain()
Pour cela, vous créerez une fonction drawCase() qui réalisera le travail pour la case passée en paramètre
Les côtés des cases doivent être affichées si le passage vers sa voisine est impossible. Utiliser pour cela libgraph->setPen(MakeARGB(20, 0, 0, 0), 0.5f); et libgraph->setPen(MakeARGB(255, 0, 0, 0), 2.0f); pour respectivement rendre le stylo translucide ou opaque.
Pour le moment, le fond des cases reste transparent : libgraph->setSolidBrush(MakeARGB(0, 0, 0, 0));

3) Commentez les créations et utilisation des cases A, B, C, D que vous venez de tester dans la fonction _tWinMain()

4) Créer un objet de type CMaze au commentaire //A faire 4) dans la fonction _tWinMain()

5) Initialisez les variables nNbCasesX et nNbCasesY en fonction des données internes de l'objet maze au commentaire //A faire 5) dans la fonction _tWinMain()

6) Dessiner chaque case du labyrinthe au commentaire //A faire 6) dans la fonction _tWinMain()

Les cases normales doivent être affichées en fond transparent en utilisant libgraph->setSolidBrush(MakeARGB(0, 0, 0, 0));
Les cases visitées doivent être affichées en fond jaune en utilisant libgraph->setSolidBrush(MakeARGB(255, 255, 255, 0));
Les cases de mauvais chemin doivent être affichées en fond rouge en utilisant libgraph->setSolidBrush(MakeARGB(255, 255, 0, 0));

Les côtés des cases doivent être affiché si le passage vers sa voisine est impossible. Utiliser pour cela libgraph->setPen(MakeARGB(20, 0, 0, 0), 0.5f); et libgraph->setPen(MakeARGB(255, 0, 0, 0), 2.0f); pour respectivement rendre le stylo translucide ou opaque

7) Réaliser un pas de l'algorithme de résolution du labyrinthe au commentaire //A faire 7) dans la fonction _tWinMain()

8) Dessiner le chemin de résolution du labyrinthe au commentaire //A faire 8) dans la fonction _tWinMain()

Pour cela, tracez un trait vert (libgraph->setPen(MakeARGB(255, 0, 192, 0), 1.0f);) le long du parcours déterminé par l'algorithme de résolution du labyrinthe.

9) Afficher les images des cases de départ d'arrivée et en cours au commentaire //A faire 9) dans la fonction _tWinMain()
L'image de la case de départ est dans le fichier start.png\n L'image de la case d'arrivée est dans le fichier exit.png\n L'image de la case en cours (fin du chemin) est dans le fichier me.png
